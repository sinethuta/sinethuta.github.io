What is MagiChess?

MagiChess is a computer chess robot that allows the user to play a game of physical chess versus a remote opponent over the internet. Weâ€™ve decided to close the gap between physical and digital chess. To do this, we plan to create a chess board that allows users to play with an AI or a remote human opponent.

Interfacing with LiChess Server and User

In order to communicate over the internet, process information, control movements and recieve user input, a Raspberry Pi is used. A Raspberry Pi is a credit-card sized computer with General Purpose Input/Output (GPIO) pins to allow for interfacing with other electronic systems. WiFi is used to connect the Pi over the internet to talk with Lichess chess server and the GPIO pins are used for controlling XY-Gantry movements and magnetic sensor input. The touch screen displays current game information and notifications as well as accepting user inputs for game and system settings.





Making Movements: The Chess Piece

Each chess piece is fitted with a magnet to aid moving and a RFID tag for unique identification. When placed on the chess board, the magnet will trigger the sensors used to monitor moves made by the user. The magnet is also used in moving chess pieces with an electromagnet under the chessboard.






Making Movements: The Gantry

XY-Gantry will be placed under the chessboard. An electromagnet is attached to the gantry and when turned on, it will attract to the magnet embedded in the chess piece. The gantry will then move, following a path sent from the path planning algorithm on the Raspberry Pi. Two stepper motors and drivers, an electromagnet, an RFID reader and a Microcontroller works together to control movements on the XY-Gantry.

Sensing Movements: The Hall Sensor Array

When the remote player makes a move, the move is sent to the path planning algorithm. It'll then use the current positions of chess pieces to plan a clear and a short path that will reflect the move made by the remote player. The program uses Greedy Best First Search algorithm and runs on the Raspberry Pi. The path is then sent to the microcontroller on gantry via UART.
